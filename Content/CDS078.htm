CDS078

Time Conversion (to display format)

 Last Revised: 10/26/06

CDS078 is called to convert either the the system variable TIM or a passed variable in the same format as TIM into two, eight character display formats.  See CDS178 for conversion from display format back to TIM format.

Calling Format:

CALL "CDS078", HMS$, HMA$, TIME

CDS078
HMS$ will be returned the time in hours:minutes:seconds (military) format such as " 1:15:22" or "23:55:38"
HMA$ will be returned the time in hours:minutes XM format such as " 1:15 AM" or "11:55 PM"
TIME Optional variable passed to CDS078 to use instead of the system variable TIM.  It can be used to format a stored time, or to format elapsed time.  The variable is in hours and fractions of an hour, just like the TIM system variable.
Example using the system time
>call "CDS078",HMS$,HMA$
>?HMS$
13:03:12
>?HMA$
 1:03 PM
>?TIM
13.06
Example using user specified time
>TIME=14.5
>CALL "CDS078",HMS$,HMA$,TIME
>?HMS$
14:30:00
>?HMA$
 2:30 PM
Example using elapsed time
>PRECISION 6

>STARTTIME=TIM
>other program logic to be timed
>CALL "CDS078",HMS$,HMA$,TIM-STARTTIME
>PRECISION 2
>?HMS$
00:00:46
>?HMA$
12:00 AM

Note: The HMA$ AM/PM format is not applicable for elapsed times.  You may need to set the PRECISION to 6 or higher in order to get an accurate measure of elapsed time.
