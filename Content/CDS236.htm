CDS236

Convert XML to Templated String

 Last Revised: 08/11/11

This called routine converts text in XML format to a templated string variable.

This program uses bbx string template field names to represent the tag names, so tag names cannot be more than 32 characters long.

Calling Format:

CALL "CDS236", ERR=XXXX, Y$, X$, S236$, XML$

CDS236 Arguments
Argument Passed To/From Description
ERR=XXXX N/A Error branch taken if:

required input X$ is null

Unable to open xml data file (if file name is specified)

XML data does not begin with "<"

Invalid xml format, missing ">"

Invalid xml format, null tag name

Invalid xml format, / contained within tag name, but not in first or last position
Y$ To Session Control Variable
X$ To Pass either a string variable containing the XML markup, or a data file name containing the markup.
S236$ From Set by CDS236 to the structure of the XML.
Field NameUsage
FLDSTotal number of unique field names in XML$.  There is a field name for each XML node.
ROOTThe root XML tag name.
TAG[FLDS]Tag names with period separators, i.e., Contents.Owner.ID would be the ID element nested inside Owner, nested inside Contents.
FLDNAM[FLDS]Field names used in XML$.  The FLDNAM is derived from the base tag name with the following exceptions.
  • If the base tag name includes a colon as in gsx:username, then FLDNAM excludes everything up through the colon
  • If the base tag name appears in the XML with different parents, then the field number is appended to the field name, i.e., NAME_2.
  • If the FLDNAM is longer than the maximum bbx template field name of 32 characters, the FLDNAM is truncated and the field number is included as the trailing characters of FLDNAM in order to insure uniqueness.  In this case, the un-truncated field name is stored in the template title field.
REPEAT[FLDS]Indicates the number of times the tag was found in the XML.  Typically used as the upper limit of loop when extracting values.  See example.
XML$ From Set by CDS236 as the XML values.  The calling program can use the information in S236$ to access the XML values contained in XML$.  Note that CDS236 does not remove line feeds ($0A$) from XML$ so that fields containing text area fields with line breaks can be processed properly.  The calling program can simply refer to FIELD(XML$,S236.FLDNAM$[FLD]) to access the value of any field.  See Example.
Example
3000 BEGIN
3005 PRINT 'CS',
3010 CALL "CDS236",Y$,"tmp/CDS700-050.xml",S236$,XML$
3020 FOR FLD=1 TO S236.FLDS
3030 IF S236.TAG$[FLD]="Contents.Key" THEN LET KEYFLD=FLD
3040 IF S236.TAG$[FLD]="Contents.Size" THEN LET SIZEFLD=FLD
3050 NEXT FLD
3060 FOR I=1 TO S236.REPEAT[KEYFLD]
3070 PRINT "Name: ",FIELD(XML$,S236.FLDNAM$[KEYFLD],I),
3080 PRINT @(34),"Size: ",FIELD(XML$,S236.FLDNAM$[SIZEFLD],I)
3090 NEXT I
