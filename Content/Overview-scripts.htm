runtime components

Scripts & Special Files

 Last Revised: 03/28/16

Dynamo Tools includes several Unix scripts that are used to start the Dynamo Tools menu processor from a terminal or PC, or as a standalone 'background' task.  Various backup and printer backend scripts are also provided.

These files are located in the $SMSDIR/bin directory.  Most of the files only apply to Unix and not to Windows installation.

File name Description
S99cdi Script to be executed at boot time to execute the bin/startup file.  This file is normally copied to the /etc/rc.d directory.  Using S99cdi is the preferred method of executing the startup script as opposed to adding an entry in the /etc/inittab file, since this file can be replace during an SMS upgrade easier than modifying /etc/inittab.
aix.sample Contains sample text that can be added to the /etc/security/login.cfg file on AIX systems to change the login message.
background Script to start a background task with the SMS environment established with the same local and global variables as a program started from the menu.
backup.aix Script to backup all files except those in an exclude list using the AIX backup command.
backup.aix.boot Script to create a bootable AIX backup including all files in the root volume group.  Preferred backup method on AIX systems when there is only one volume group.
backup.cpio Script to create a complete backup using cpio.  Script re-reads tape after backing up and counts files on tape.  Sends email with backup status so operator is informed of backup success or failure.
backup.tar Script to create a backup using tar.  This is an obsolete script which has been replaced by backup.cpio.
backup.exclude Another variation of backup.aix.
catslv Prints a text file to slave printer on terminal or PC that has a terminal environment variable $TERM of wy60, wy160, ansi, ansimono, ansicoe, vt100, termmono, termcoe, or facetwin.
cdi.fwk A facetwin keyboard mapping file that enables Ctrl+PageUp and Ctrl+PageDn.
cdikeys.cmd A keyboard mapping for use with Century Software terminal emulators.
exlogo.ico A Windows icon file, used as the default icon when installing Dynamo Tools.
config.sample Samples with explanation of typical entries in the BBx configuration file config.bbx.
cmpdel Script that compare files in two directories and deletes all files in one directory that are the same in both.
ctty Used in place of stty for serial terminals connected to the 16 port Remote Asynchronous Node (RAN box) on AIX systems.
dalzr126.prs Wordperfect printer definition file for a laser printer compatible with VSIFAX.
dialcvt.example An example of a bbx based routine to perform telephone number conversion including long distance prefix and suffix needed for VSIFAX.
dt_lst

This script lists compiled bbx and bbj programs and performs indenting of IF/THEN/ELSE commands. It also inserts blank lines before line labels and REM statements. The output file is compatible with pro5cpl and bbjcpl making dt_lst suitable for use with source code control scripts. Both pro5 and bbj must be installed on the server. This functionality is equivalent to running the Program Utility (SMU) Convert Programs to Text option.
Usage: /u/CDI/bin/dt_lst PGMFIL DIROUT
PGMFIL is a single tokenized/compiled program.

DIROUT is the directory where the listed program will be placed.
faxascii.prs Wordperfect printer definition file for an ascii printer compatible with VSIFAX.
faxrnotify Receive notify script used by VSIFAX to format inbound faxed images for use with the broadcast fax options of the Dynamo application software.
faxtnotify Transmit notify script used by VSIFAX to populate the Fax Log & Options (PFM) and to send a message to a user in case of an unsuccessful fax.
ip.sh Script used at login to determine the ip address of the login device. Formerly named ip but renamed to avoid conflict with linux ip command.
l Shortcut for the unix command: ls -l.
lphp Script for use in AIX systems to print text file to a Hewlett Packard or other printer that requires insertion of carriage returns.
magic Text file used to modify the /etc/magic file so that the unix file command can identify bbx programs and data files.
mas90 Script to start the MAS90 application.
mods Script to display programs with specific naming convention in date modified sequence in operator specified directory.
p Shortcut to the unix more command.
profile Script that is copied to .profile in the home directory of each user created with Add/Maintain Operator Information (SMC).  This script runs either profuser, profroot, or profcdisc.
profcdiisc Script that is used by the cdiisc login id used for unattended file transfer using Inter System Communications.
profroot Script run by profile for the root user.  Provides a root option menu for popular options run as root.
profuser Script run by profile for normal users. 
prt1 Simple printer backend script that incorporates bbx lock file logic to permit simultaneous direct printing and spooling.
prt1.sco Modified SCO printer backend script suitable for serial printers.
prt1.scofct Same as above without locking mechanism suitable for printers attached to PC's running facetwin.
restartBLM.sh Script to restart the Basis license manager so license log files do not grow indefinitely.  Typically executed by root's cron weekly.
recoverall.sh Script for linux servers to run the Basis mkrecover utility on all eligible files in and below a specified directory. Will spawn multiple sessions, 10 by default, and processes the largest files first to reduce total elapsed time.
savefiles Script to bundle certain key files into a tar bundle.
setiscport Script to maintain serial port settings for printing to serial ports on SCO systems.
startup.sample Example startup script executed at boot time to execute setiscport, start command processor, etc.
termcap Modification of the Basis terminal definition file with significant enhancements for terminal types: wy60, wy160, facetwin, termcoe, ansicoe, and ibm3151.
terminfo.cdi Source file for terminfo definitions.  Used during SMS installation to update terminfo definitions used by vi and other unix programs.
termset Script used during normal login to determine terminal type.
termset.txt Text file used by termset containing list of popular terminal types.
vsifax.config.txt Sample text for VSIFAX configuration files.
wordperf50 Script to start the Unix version of WordPerfect 5.0.  Normally executed from OSP menu.
wordperf51 Script to start the Unix version of WordPerfect 5.1.
wordperf8 Script to start the Unix version of WordPerfect 8.
wp.fwk Facetwin keyboard mapping to enable WordPerfect 5.0 for Unix keys including Ctrl+End, Ctrl+LeftArrow, Ctrl+RightArrow, Ctrl+Enter, Ctrl+Backspace, Ctrl+PageDn, Shift+Tab, F11, and F12.
wp2.fwk Same as above for WordPerfect 5.1.
wpkeys.cmd Keyboard mapping file for Century Software terminal emulation to enable WordPerfect keys.
