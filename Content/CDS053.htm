CDS053

Set Display Color

 Last Revised: 07/12/11

CDS053 is called to print and/or return the display colors using the Dynamo Tools color number.  You may specify the color number to use, or use the colors specified by user or company code.

Calling Format:

CALL "CDS053", Y$, SYSD02, COL$, PAR$, COLPRN$

CDS053
Y$ terminal control variable, used to identify terminal type
SYSD02 Earlier version of CDS053 used data file SYSD02 to store operator color preferences.  Although this file is no longer used, the placeholder remains on the ENTER list as to not require modifications to programs that call CDS053.  You can pass it as 0, or any value, as it is no longer used by CDS053.
COL$ Typically passed as null to use the colors specified for the user and company, or default colors for the company, if the user has no preference specified for the active company code.

 

If COL$ is passed to CDS053, then it should be 10 color numbers, each 3 digits long.  The color number is a value from 1 to 128 and describes both the background and text color for in both bold and dim intensities.  You can see the color numbers in the screen shot below.

 

The first color number is an odd numbered color and is used for bold intensity text for the primary window.  The dim intensity text is always the next higher color number.  This is determined by bbx and cannot be changed.

 

The next 3 color numbers describe the colors used for the F1 Hotkey window.  The first color is used for the window border, the second color is used for the window contents, and the third color is used as the highlight row.

 

The next 3 color numbers describe the colors used for F2 List Selection windows.  The colors describe the window border, window contents, and highlighted row.

 

The final 3 color numbers describe the colors used for the F3 Help Window.  The colors describe the window border, window contents, and highlighted row.
PAR$ Can be passed as R to indicate that the color should be returned to the calling program, but not printed.  Useful when you need the color string for an argument to the 'WINDOW' mnemonic.
COLPRN$ This variable is set by CDS053 and returned to the calling program as a mnemonic that can be used to set the default colors.  You can set the default colors using PRINT COLPRN$,
