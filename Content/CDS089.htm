CDS089

Enter Terminal ID / IP Address / PC Name

 Last Revised: 11/11/07

This routine is used for entries of a normalized Terminal ID, IP Address, or PC Name.

CDS089 is a C type external routine that can be used with file maintenance, or called by an application.

CALL "CDS089", MINVAL, Z2, Z5, Z6, Z0$, Z, Z$, Y, Z8$

MINVAL Minimum entry, typically 1 for Terminal ID's, 7 for IP addresses, and 1 for PC Names.
Z2 Maximum value: 4 for Terminal ID's, 15 for IP Addresses & PC Names
Z5 Entry Column
Z6 Entry Row
Z0$ Prompt
Z will be passed to calling routine as length of normalized entry
Z$ value
Y will be passed as CTL value
Z8$ P - prompt mode
Q - inquiry mode

other CDS069 string or file validation containing { and }

In normal format, the terminal ID is four characters long. The first character is a 'T' and the last three are numeric characters, zero filled. See examples.

T0 T000
1 T001
T10 T010
TA T010
17 T017

For networked terminals, the terminal ID varies with each login, and so is not suitable for information linked to a particular device. Instead, the IP address should be used. The IP address is most often fixed for local devices, but also can vary with devices connected via the internet.

To enable CDS089 to permit entry of an IP address, set the maximum length to 15. The operator will be able to enter either a terminal ID or an IP address.

To require an IP address, but disallow terminal ID's, pass a minimum value to 7.

There are some circumstances where a blank field is permitted, in addition to any normalized ID. To tell this routine that a blank field is also permitted, simply specify the minimum value of the variable data dictionary to 0. Any other value indicates to the routine that only a valid normalized terminal ID is permitted.
