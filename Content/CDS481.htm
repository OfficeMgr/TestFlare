CDS481

Elapsed Time Entry HH:MM format (T Type)

 Last Revised: 05/19/09

This T type called routine can be used for entry of elapsed time stored in a numeric variable in the bbx TIM format, i.e., hours and fractions of an hour.  CDS481 is typically specified as the program name to call for time entries in the Data Dictionary.  The time can be from zero to 23 hours and 59 minutes.

It is strongly recommended to use a B type template field to store the data.  This will insure sufficient number of digits to the right of the decimal to convert all minutes to fractional hours.  If using an IOLIST where you cannot use a B type template field, then use N(12*).

The routine is flexible and allows many different entries such as:
10 (minutes not entered)
115 (no colon or period entered)
1:15
1.15 (use period instead of colon)
01:15 (leading zero entered)

Calling Format:

CALL "CDS481", S039$, REC$, X100$

CDS481
S039$ Templated parameters for entry
REC$ The data structure containing the numeric time field, i.e., NFIELD(REC$,S039.FLDNAM$,S039.INDEX)
X100$ Standard T type routine parameters including X100.ACTION$
Example
1010 BEGIN
1020 PRINT 'CS',
1030 CALL "CDS276","Elapsed Time",COL
1040 DIM REC$:"ELAPSED:B"
1050 CALL "CDS041","CDS039",S039$,"YY"
1060 LET S039.FLDNAM$="ELAPSED",S039.INDEX=1,S039.COL=COL,S039.ROW=2
1070 CALL "CDS041","CDX100",X100$,"YY"
1080 CALL "CDS481",S039$,REC$,X100$
1090 CALL "CDS279",REC$
