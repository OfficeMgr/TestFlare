CDS079

File Verified Entry (OBSOLETE)

 Last Revised: 05/16/99

CDS079 is a called input routine to be used instead of CDS069 when the keyboard entry is for a key to a masterfile such as a customer, item, or vendor number. In addition, this routine permits selection of the masterfile key via an auxiliary sort file.

For example, this routine permits the operator to enter either a customer number if known or part of the alpha sort name for the customer. The routine will display the second field of the masterfile (customer name for this example) on the screen and allow the operator to accept that customer or scroll on to the next customer on file with the same alpha sort that was entered.

 

NOTE: This program is obsolete and should not be used for new code. It is provided for compatibility with older systems. The same functions provided by CDS079 are included in the file verified options of CDS069.

The variables passed to CDS079 are very similar to CDS069.

Calling Format:

CALL "CDS079", Z1, Z2, Z5, Z6, Z0$, Z1$, Z, Z$, Z2$, Z3$

This routine applies only to string variables.

CDS079
Z1 minimum length of acceptable entries
Z2 maximum length of acceptable entries
Z5 screen position for input
Z6 screen position for input
Z0$ the prompt message with two minor differences.

The calling program should not include the word ENTER or the + symbol. If CTL IV is a valid response, then prefix Z0$ with a 4, i.e., Z0$="4CUSTOMER NUMBER".
Z1$ the entry type excluding numeric masks
Z will return the length of the entry
Z$ will return the string variable
Z2$

Z2$ contains a number of data elements which identify the characteristics of the auxiliary sort files:

Byte

(1,1) - Device number of masterfile, opened by calling program

(2,2) - Horizontal screen position for display of 2nd field in masterfile, such as the customer name

(4,2) - Vertical position

(6,1) - Device to be used for the auxiliary sort file
  a) if opened by calling program to the auxiliary sort file then the routine will use and leave the file open
  b) if opened by calling program to some other file, the routine will close the specified device, open as the auxiliary sort file, then close and reopen the original file
  c) if the device number was not in use, the routine will use the specified device for the auxiliary sort file, then closes the device

(7,2) - The length of the auxiliary sort entry such as the alpha sort name

(9,6) - Auxiliary Sort File Name which may have XX as the first two characters (15) - The prompt for the auxiliary sort data element such as ALPHABETIC SORT NAME (this variable may be any length, within the confines of the message line)
Z3$ the prefix to the auxiliary sort file, if any
Example

0290 CALL "CDS079",1,6,17,2,"4CUSTOMER NO, 'D' TO CHANGE DATE ","R",0,A$,"102037 0290:08XXUS10ALPHA SORT NAME",""

This example would generate the prompt:

ENTER CUSTOMER NO, 'D' TO CHANGE DATE, 'CR' TO ENTER ALPHA SORT NAME OR CTL IV
