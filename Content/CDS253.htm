CDS253

Return Displayable Field Definition

 Last Revised: 03/26/09

This called routine is used to return the field definition string for a template field.

CALL "CDS253", DIMTPL$, FLDNAME$, FLDINFO$, PAR$, FATTR$

CDS253
Variable To/From Usage
DIMTPL$ To a dimensioned template variable
FLDNAME$ To the name of a field within the template
FLDINFO$ From output from CDS253 and will contain the field name and field definition information. See examples.
PAR$(1,1) To Y indicates that the variable length indicator * and *= should be included in FLDINFO$. Default is Y.
PAR$(2,1) To Y indicates that a $ should be included in FLDINFO$ if the field is a string. Default is N.
PAR$(3,1) To Y indicates that the output FLDINFO$ should also include the user defined section of the template. Default is N.
FATTR$ From output from CDS253 and is a templated definition of the field attributes

The template for FATTR$ is:
TYPE:U(1)
FLAG:C(1)
TERM:U(1)
RPT:U(2)
FLDNUM:U(2)
OFFSET:I(2)
LGTH:U(2)
USR:C(1*=)

Refer to FATTR() in the BBx documentation for additional information.
Examples

Assuming a template: DIM DIMTPL$:"NAME:C(30),SALES[12]:B,CODES:C(5*)"

Call "CDS253"... FLDINFO$
DIMTPL$,"NAME",FLDINFO$,"","" NAME:C(30)
DIMTPL$,"NAME",FLDINFO$,"NY","" NAME$:C(30)
DIMTPL$,"SALES",FLDINFO$,"","" SALES[12]:B
DIMTPL$,"CODES",FLDINFO$,"NN","" CODES:C(5)
DIMTPL$,"CODES",FLDINFO$,"YN","" CODES:C(5*)
DIMTPL$,"CODES",FLDINFO$,"YY","" CODES$:C(5*)
