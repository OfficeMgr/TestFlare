CDS173

DUMP in html format

 Last Revised: 05/26/15

This routine performs a bbx DUMP and formats the output suitable for display in a web browser, email body, or email attachment.  It is automatically invoked by the Dynamo Tools error routine CDS063, but can also be called to assist debugging any bbx application including background tasks and Dynaweb applications.

CDS173 lists the program called at each level, and then for each call level, lists variables alphabetically within type: Numeric Variables, Numeric Arrays, String Variables, String Arrays, and for bbj applications, object variables.  String Variable that have an associated template are shown with the template structure.  In order to minimize the output, numeric array elements that are zero, and string array elements that are null, are omitted from the output.  CDS173 also lists all open files and pertinent file information.

Note that in order to limit the size of the output, the contents of string variables are truncated to approximately 4000 characters.

Global variables are also shown except those used to store templates and other Dynamo Tools global variables that begin with "SMS_".

Calling Format:

CALL "CDS173" { DUMPFILE$, HDR$, LOGMSG$ }

When debugging Dynaweb applications, simply add CALL "CDS173" at the appropriate place within the program logic.  A web page containing the formatted DUMP will be displayed. You can also do that from console mode while debugging a program.

CDS173 Direction Usage
DUMPFILE$ To/From Optional file name where formatted dump in html format will be stored. If not passed, the default file name of tmp/dump-nnn.htm will be used. Can be used to archive file for logging, emailing, etc.
HDR$ To Pass as a non-null html text to include this text in the heading of the dump file before the table containing the formatted dump. When used this way, CDS173 will not display the dump file contents in a browser window.
LOGMSG$ To LOGMSG$ is displayed to the user at the bottom of the screen during the dump process. Not applicable for background or Dynaweb sessions.
Example

0100 LET A=5,B=10
0110 DIM AMT[10]; LET AMT[1]=1,AMT[5]=5.555,AMT[9]=-1234.56
0120 DIM STR$[5:8]; LET STR$[6]="Value for Element 6",STR$[8]="Element 8"
0130 LET ID$="This is a test"
0140 CALL "CDS252",CHAN,FID$,FIN$
0150 CALL "CDS095",SM03,"CDSM03","YNY",SM03$
0160 READ RECORD (SM03)SM03$
0170 ESCAPE

RUN

CALL "CDS173"
