Launching Dynamo Tools

 Last Revised: 01/14/16

On Linux, AIX, and other Unix operating systems, you can launch the Dynamo Tools menu immediately upon login using the files described below.

Pointer File

Dynamo Tools uses a pointer file located in a fixed location on all systems at /usr/cdi/.config. This file is required and contains the following environment variables. You may also add any additional variables that you may require. The environment variables should also be 'exported' so that they are available to sub shells.

Variable Name Description Examples
SMSDIR Base Dynamo Tools directory
Can include data server syntax when Dynamo Tools are installed on another server		 /u/CDI
/<bbds>/u/CDI
BBXDIR Directory where either the pro5 or bbj executable is located /u/basis/pro5
/u/basis/bbj/bin
TMPDIR Optional folder indicating where temporary files should be stored /u/CDI/tmp
SMSTMP Required when SMSDIR includes data server, used to indicate directory for temporary files on local server /u/CDI/tmp
SMSBIN Required when SMSDIR includes data server, used to indicate location of profile and related launch files on local server /u/CDI/bin
SMSPGM Optional full path to program name required when using the standard Dynamo Tools profile to run a menu or other launch program other than CDS000. /usr0/PROG/START.MENU

 

Profile File

On Unix like systems, commands in the profile file in the user's home directory are executed upon login. When using the bash shell, the profile file is named .bash_profile and .profile for most other shells. Dynamo Tools provides file $SMSDIR/bin/profile which is typically copied to the .profile and .bash_profile files in each user's home directory, both when creating a new user and when updating Dynamo Tools.

The Profile File will verify that the necessary environment variables have been set, and the required directories exist, then will pass control to another command file based on the login ID.

In most cases, you can use the profile file provided by Dynamo Tools without modification. There are also options built in to theses files to include execution of custom commands required for your installation. You can also disable the use of the Dynamo Tools profile files by setting the "Use Dynamo Profiles" parameter to "N" in Dynamo Configuration Maintenance (SMC). This will bypass copying the Dynamo Tools profile file to the user's home directories during a Dynamo Tools update.

If you are not using the Dynamo Tools profile file, but are using the Dynamo Tools profuser file, then your profile file must include setting the following environment variables.

Variable Usage Example
SMSDIR Base Dynamo Tools directory /u/CDI
SMSBIN Directory containing other launch files including profuser, id, termset, termset.txt /u/CDI/bin
SMSTMP Directory where temporary files are to be stored /u/CDI/tmp

 

Custom Launch Routine

The Dynamo Tools Profile File includes logic to execute a command file with your custom commands and logic. This file must be named .altsmsprofile. This file can either set variables and return control to the calling routine, or execute custom startup logic, and not return control to the calling routine.
