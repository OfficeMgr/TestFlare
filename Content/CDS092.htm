CDS092

Rename File

 Last Revised: 02/16/13

This called routine should be used to rename a file or program.

Calling Format:

CALL "CDS092", ERR=XXXX, CHAN, NEWNAME$

CDS092
ERR= The line to branch to if the rename is unsuccessful.  Possible error conditions include error 12, if file NEWNAME$ already exists, or error 0 if the file opened to CHAN is not currently locked, and cannot be locked.
CHAN the channel of the file to be renamed, opened by the calling program.  The file opened to CHAN will be closed by CDS092.  So the application will need to re-open the file to the new name if necessary.
NEWNAME$ the new file name
 

Can also be passed as null, in which case CDS092 will rename the file using the standard naming convention for archived files.  First a suffix of .YYMMDD is added, then alpha suffixes are added to the date when necessary to force a unique file name.  NEWNAME$ will be returned as the file name assigned.

If NEWNAME$ does not contain a slash, a backslash, or a colon, then CDS092 assumes you want to rename the open file to NEWNAME$ without changing the directory.

This means that if you want to rename a file from a subdirectory to the current directory, you must include ./ on the front of the new file name.

If NEWNAME$ does contain a slash or backslash, but does not start with a slash or backslash, or DOS drive letter, then CDS092 assumes the new name to be relative to the current directory.
