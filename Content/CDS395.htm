CDS395

.dbf File Open and Setup

 Last Revised: 02/26/09

This program can be used to open a string data file in the standard .dbf format. This format is used by some data base programs such as Dbase.

In addition to opening the file, CDS395 will return a template that you can use when reading the file.

CALL "CDS395", ERR=XXXX, CHAN, FILNAM$, PAR$, REC$

CDS395
CHAN a channel that the file will be opened to. It may be passed as zero, in which case an unused channel will be used and returned for use by the calling program.
FILNAM$ the name of the .dbf file to be opened.
PAR$ currently only has one character, which is passed to CDS095 as the parameter that indicates whether a message should be displayed if the file cannot be opened
REC$ the templated variable to be used when reading the file

READRECORD (CHAN,SIZ=LEN(REC$), END=XXXX)REC$
IF REC$=$1A$ GOTO EOJ; REM "dbf files may have trailing $1A$ at end of file

The basic format of a dbf file includes:

  • header information
  • field information
  • optional information
  • data records

After calling CDS395, the file will be positioned at the start of the optional information. If there is any optional information, then you will have to experiment to determine its length in order to position the file at the first data record.
