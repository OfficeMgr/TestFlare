CDS082

Display Text

 Last Revised: 02/25/05

This program is called when text from a documentation text file is to be displayed on the terminal.

Calling Format:

CALL "CDS082", ERR=XXXX, Y$, T$, F$

CDS082
Y$ the terminal control variable, may be passed as null if not available. Used to determine if the terminal has lower case capability.
T$ the Text ID of the text to be displayed in the format: CCPPPPVP when using documentation text from file XXSD56 where CC is company code PPPP is program name V is VDT suffix P is pass parameter CCTTTTTT when using text from any other file such as XXED80 where CC is company code TTTTTT is text ID
F$ is only required when not using the documentation file XXSD56 and should contain the four character file ID such as "ED80"

The most likely error with this routine is an error 31 due to insufficient memory to read the text file. Since the characters per record of the text file is large (1920 characters), there must be more than twice that amount of available memory or approximately 4000 characters of DSZ to read the record. Should an error 31 occur, CDS082 will display a message that there is insufficient memory.
