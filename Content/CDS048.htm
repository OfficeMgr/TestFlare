CDS048

Enter Telephone Number (X)

 Last Revised: 11/10/12

This routine is called for telephone number entries. It supports telephone numbers in these formats: AAA-EEE-NNNN 1-EEE-NNNN EEE-NNNN The telephone number is entered without any dashes, and the routine will return it formatted with dashes.  Alpha characters in the telephone number such as 800-IBM-4AIX are not supported.  A telephone type and extension are also supported.

See also CDS268 which supports longer foreign phone numbers and does allow alpha characters.

The data to be entered is dependent on the length of the data field Z$. If Z$ is 12 characters long, then only the telephone number will be entered. If 13 characters long, the additional character is used for a telephone type code. If 17 characters long, the additional 4 characters is used as an extension.

This program performs validation checks on phone numbers.

Calling Format:

CALL "CDS048", Z1, Z2, Z5, Z6, Z0$, Z7, Z$, Y, Z8$, FMT$[ALL]

CDS048
Input Variables
Z1 minimum length of data to be entered
Z2 maximum length of data to be entered
Z5 horizontal screen position
Z6 vertical screen position
Z0$ operator message
Z7 passed as zero
Y passed as 0; set to CTL
Z8$ action code
DIS - display without stripping spaces, as would be done for a column of phone numbers
DISFMT - display flush left, stripping area code if not present. Adding "Ext: " to extension if no phone#.
FMT - format only, no display. See FMT$[ALL] below
DRV - entry suitable for driver, no area code, type, or ext - all other used for entry of phone#.
If first character is P, use prompt window.
If 3 or more characters long, Z8$(2,2) is used to identify the state code which is used to allow unformatted phone numbers for foreign locations, i.e., no area code and xxx-xxxx format restrictions.
Output Variables
Z$ formatted telephone#
Z7 length of telephone number
FMT$[ALL] [0] - output without stripping leading spaces
[1] - leading spaces deleted

Optional Telephone Type Codes

blank Any
W Work
F Fax
