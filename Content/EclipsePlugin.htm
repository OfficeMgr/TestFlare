Developing with Eclipse

Last Revised: 08/02/16

 

The Eclipse Integrated Development Environment (IDE) is an open-source platform that allows developers to write code in a customizable workspace. Though the IDE is JAVA-based, the Eclipse community has created plug-ins for many programming languages that assist with automatic code-completion, debugging, syntax checks, and more. With customizable views—called perspectives in Eclipse—the developer also has great control over what can be seen at-a-glance.

Excellware has developed a Dynamo Tools plug-in for Eclipse. The purpose of the plug-in is to make BBx/BBj coding with Dynamo Tools and custom company libraries more efficient while providing the ability to work in a modern, feature-rich, and customizable development environment.

For those new to Eclipse, helpful tutorials can be found at tutorialspoint.com, vogella.com, and, of course, the Eclipse community.

Installation

Please perform the following steps in order. Click on a step for more information.

Install Java Development Kit (JDK)

Get the latest version of JDK from Oracle's website. For PC, be sure to select the version that matches your system (i.e 32-bit(x86) or 64-bit(x-64).

 Install BBj version 16

Install the latest version of BBj 16 from Basis International's website for Windows or UNIX/Mac. (Barista and AddonSoftware do not need to be installed).

Note: A valid BASIS license is necessary or use of BBj will be confined to a 30-day trial. Licenses can be applied manually on a local machine by designating the hard drive location of a license file from Basis. Alternatively, the Basis License Manager (BLM) can be installed to dispense licenses to users as needed. Use the BLM option to designate a local or remote server location. For help with locating the BLM server or obtaining an existing license during BBj installation, consult your company's overseer of BASIS products.

Install Eclipse IDE version Mars 2 for Java EE Developers

There are many previous versions of Eclipse but currently the plug-in only works with Mars 2. There also are many "flavors" of Eclipse to compare, but Eclipse for Java EE Developers is recommended in this case for its known compatibility and robust list of included features.

  • There are two ways in which Eclipse can be added to a machine. These methods are the same for Windows, Mac, and Linux.
    • Download the latest version of Eclipse Mars 2 package for your OS from the Eclipse website. Extract the contents of the zip file to a hard drive location where you would like Eclipse to reside and run from. Run Eclipse by opening the Eclipse application file within the extracted folder. Note this option does not install Eclipse and removal of Eclipse involves only deletion of this extracted folder and the workspace folder that is designated upon first running the program.
    • Using the Eclipse installer will automate the installation of the latest version of Eclipse. However, the Dynamo Tools plug-in has currently only been tested with Mars 2, which is not the most current version. Therefore, the first method of adding Eclipse to your machine should be used.
 Install the Basis Developer Toolkit (BDT) plug-in version 16.xx for Eclipse

There are several different Basis plug-ins for Eclipse. You can visit the Basis Eclipse Plug-Ins page to peruse them. However, the only one that is required here is the BDT CodeEditor plug-in, the URL for which is already provided in step 3 below.

  1. Launch Eclipse.
  2. In Eclipse’s main menu, select Help > Install New Software...
  3. At the top right, click [Add...] to open Eclipse’s Install window. This window needs two pieces of information: a Name of your choosing that identifies the plug-in, and the URL http://plugins.basis.com/bdt/16xx. Next Click OK
  4. You should now have one collapsed entry in the list field of Eclipse’s Install window

  5. Ensure that the “Contact all update sites during install to find required software” box is checked

  6. Check the box next to the BDT entry in the Name field and click [Next >]
  7. Review the Install Details list shown and click [Next >].
  8. Follow the remaining instructions as appropriate based on the plug-in you are installing.
  9. Once the Install window completes successfully, you may be prompted to restart Eclipse. You are able to install the Dynamo Tools plug-in using the next steps before restarting.
 Install the Dynamo Tools Plug-in for Eclipse
  1. Launch Eclipse
  2. In Eclipse’s main menu, select Help>Install New Software...
  3. At the top right, click [Add...] to open Eclipse’s Install window. This window needs two pieces of information: a Name of your choosing that identifies the plug-in, and the URL http://dl.excellware.com/plugins/16/. Click OK
  4. You should now have one collapsed entry in the list field of Eclipse’s Install window
  5. Ensure that the “Contact all update sites during install to find required software” box is checked
  6. Check the box next to the Dynamo Tools entry in the Name field and click [Next >]. 
  7. Review the items to be installed and click [Next >] 
  8. Follow the remaining instructions as appropriate based on the plug-in you are installing.
  9. Click OK when prompted to restart Eclipse. If not prompted, exit Eclipse manually and restart the application for changes to take effect
  10. Dynamo Tools will now appear as a menu option in Eclipse and the Dynamo Tools Helper pane will be visible on the right

 

Getting Started

After installation steps are performed:

Open the BDT perspective
  1. Open the BDT plug-in perspective by clicking the Open Perspective icon in the top right of the application. If this icon can not be found, navigate to Window>Perspective>Open Perspective>Other...
  2. Select BDT and Click OK
 "Export Data Dictionary to Eclipse" from within Dynamo

For a company's custom data dictionaries and called routines to be made available in Eclipse, this option must be set to Y. This option is found in Dynamo under SMC>Company Information Maintenance>Development Projects.If you do not see Development Projects, press F4 after entering Company Information Maintenance

 Add custom company libraries to Dynamo Tools Plug-in

The two-lettered Dynamo company code(s) for your company should be added to customize the code-completion suggestions for custom data dictionaries and called routines. To do this,

  1. In Eclipse, navigate to Windows>Preferences
  2. Choose Dynamo Tools in the column on the left
  3. By default, CD will be the only available code in the center pane. On the right, choose Add, enter your Dynamo company code (or codes if your company has more than one) and click OK.
  4. Click OK. Your company code should now appear below CD in list of Companies Libraries
  5. Exit the Preferences window
 Enable or disable Y$ as a session control variable

The use of DT_Y$ is disabled by default for any given project. However, it can be enabled on a per-project basis. If your company uses Y$ as a session control variable, enabling this option can decrease the chance of conflict with existing company software.

  1. After opening a BBj project, find it in the BDT Explorer view on the left of the Eclipse window. (click thumbnail below)
  2. Right-click and select Properties
  3. In the resulting project Properties window, select Dynamo Tools on the left
  4. Select your company code from the drop-down list
  5. Check the box Use DT_Y$ for the project
  6. Click OK

 

 

Coding Features

The following is a list of keyboard shortcuts to use from the code editor. Note that the description of these shortcuts, as well as their corresponding key combinations, appear in the Eclipse Dynamo Tools menu.

Description Usage

Content Assist

(Ctrl+Shift+Space)

Used for the suggestions and auto-fill features of the plug-in. This feature is contextual so it will only make suggestions that work with what is already entered into the code editor at the cursor position. If nothing is entered on the editor line at the current cursor position—such as at a blank line—a list of all available entries will be presented with this keyboard shortcut. In either case, a search box is also presented to easily find what is needed.

Pressing Ctrl+Shift+Space at: produces:
a blank line in editor

a search box in the Dynamo Tools Helper section where you can search for all available data dictionaries and called routines. Searches can be by name or description.
a partial CD entry in editor

a search box within the editor populated with suggestions. Here, typing the name of the called routines narrows the search results to the name of the function as well as a short description. You can also search by description/keyword of the function to narrow the list of called routines.
a partial dd entry in editor

a search box in the Dynamo Tools Helper section that is populated with suggestions for all available data dictionary entries. Searches can be made by name of data dictionary or description/keyword.

Go to Dynamo Tools Helper Window

(Ctrl+Shift+Z)

Instantly moves the cursor from the Editor to the Helper view to perform a quick search of all routines and data dictionaries. If what you were looking for does not appear here, make sure you have imported your company's routines and dictionaries as outlined in the Getting Started section above.

Show Routine Help

(Alt+F1)

Placing the cursor on a called routine and using this shortcut to open a detailed documentation page for that routine in a browser.

BBx Syntax

(Alt+C)

The plug-in will initiate a check for BBx syntax errors
