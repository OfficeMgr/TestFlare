inter-system communication

Batch Process

If the application software uses CDS036 to record all file writes and removes, then the files and processes below are used to transfer data between a headquarters and branch system, or between a production and a failover or Disaster Recovery site.

File Name Type Usage
XXIS00 MKY ISC Files to log. Defined and maintained using Files to Log Maintenance (ISC).  Read by CDS036 to determine if file activity needs to be logged on each system.
XXIS01 MKY ISC Logged records - CDS036 records file name, primary key, and a write/remove indicator in this file.  Records are removed from this file by CDC950 as the records are transferred to the ISC/XXIS10.zzz file.
XXIS02 MKY ISC System Information - Defined and maintained using System Information Maintenance (ISC).  Contains system information including selection code for each location.
XXIS01_X MKY The last 9 XXIS01 files are retained with a _1 through _9 suffix with _1 being the most recent.  The rolling and purging of the oldest file takes place by CDC950 in preparation for transmission.
ISC/XXIS10.zzz SER This file contains the file names, records to be written, and keys to be removed from files on the target system with system number zzz.  These are the files that are transmitted to a remote system. Note that these files are located in the ISC directory.  The CDC950 program defines, and writes to these files.  The ISC script initializes these files after successful transmission.
ISC/XXIS20.zzz SER This file contains the data received from system zzz.  The file is created or appended to by the ISC Queue.  Records in the file are processed by the CDC960 program.
ISC/XXIS20_X.zzz SER The last 9 ISC/XXIS20.zzz files are retained and renamed with an _X suffix, with _1 being the most recently received file.
