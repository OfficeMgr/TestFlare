CDS065

Filelist Utility

 Last Revised: 10/10/13

A Filelist is simply a list of file names stored in a string file.  This utility is not normally called from application software.  It is designed to permit entry of file names for processing by another program such as the Program and Disc Utilities on the SMU menu. The list of file names can also be stored for later use in the FLIST folder.  This program is also available directly from the SMU menu.  The Filelist utility can scan directory entries to include matching files in the list.  Filtering by file type, file date and file size is also available.

Calling Format:

CALL "CDS065", Y$, PATH$, TYPES$, ROW, CHAN, {FILSIZ}

 

CDS065 supports directory searches on all operating systems including file systems accessed by a data server.

CDS065 also supports operating system commands find and locate when available.

CDS065
Y$ In/Out standard terminal control variable
PATH$ Input Optional path where files are located
TYPES$ Input

An list of zero or more file types to be included.  Null indicates any file type can be included in the filelist, otherwise file types not included in the list will be excluded.

PGM listable program saved in current version of business basic
PGn listable program, saved in version n of business basic, i.e., PG4, PG5, PG6
EPG encrypted program
EPn encrypted program, saved in version n of business basic
MKY Multi-keyed
SKY Single Key
DIR direct
SRT sort
IND indexed
SER serial
STR string
ROW Input Top ROW number of display that CDS065 will use for heading
CHAN In/Out Channel number of file list file.  If passed as zero, CDS065 will assign CHAN as an available channel number.  If passed as non-zero, then CDS065 will close any file opened to this channel, and open the file list file to channel CHAN.
FILSIZ Input Optional
FILSIZ=0Include all files when searching
FILSIZ>0Include files of FILSIZ or greater when searching
FILSIZ<0Include files of -FILSIZ or smaller when searching

Note that when entering single file names, CDS065 will display the file description and type to the right of the file name. For maximum performance, this information is not displayed when scanning a directory or when using the locate option.
