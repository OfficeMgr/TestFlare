Program Editor

Command Summary

 Last Revised: 06/16/15

The normal BBx prompt is the "greater than" symbol ">".  The prompt used when running the Program Editor is the "right parenthesis" symbol ")". At the ")" prompt, you can enter any valid BBx statement, or any Program Editor Command.  To do this you simply type the command the same as you would a BBx statement.

PRINT 2+3

IF LEN(X$)>20 PRINT X$

To add or replace a program statement, start with a line number, just like you would in BBx.

0320 IF GL012.GLCODE$="FRT" THEN LET FRTGL$=GL012.GLACCTNO$

Program Editor Commands are not case-sensitive.  Arguments within { } are optional.

Program Editor Commands
'lineno or 'linelabel Edit program statement at lineno or linelabel. 
S{lineno} or S linelabel Display one screen of formatted program statements beginning at lineno/linelabel (if specified).  Note that a space is required after the "S" when using a line label.
L{lineno} or L linelabel Similar to the S option except the program statements are not indented or formatted in any way.
T{xxxx{,yyyy}}\search_text Search lines xxxx through yyyy (if specified) for search_text.   Search_text is not case sensitive.  Note you can also use a forward slash / separator which is useful when searching for back slashes \.
T{xxxx{,yyyy}}\search_text\replacement_text Search and replace all occurrences.   Search_text is case sensitive when replacing.  Note you can also use a forward slash / separators which is useful when replacing back slashes characters \.
T{xxxx{,yyyy}}\search_text\replacement_text\V Same as above, but prompt operator for each replacement.
E{xxxx{,yyyy}}\search_text Edit each line containing search_text. Forward slash can also be used as a separator.
Mxxxx{,yyyy}\pgm{\zzzz}

Merge lines xxxx through yyyy from program "pgm".   If zzzz is specified, the merged code will be renumbered so the first line merged is zzzz.  Line numbers within merged statements will also be changed.  Watch for line numbers that refer to lines outside of the merged section such as IOL= which may need to be edited manually.

Note: When running bbj, reference to line numbers are not currently being changed automatically, and must be changed manually.
Cxxxx,yyyy Concatenate line yyyy onto line xxxx and separate with semicolon.
F Searches program for invalid template fields.  Positions cursor after period to allow F2 key to select and paste correct field name.
P Display program information
SAVE The first statement of the program must be a remark.   The SAVE command changes the date on this first statement to today, prompts the user to enter a revision note, and saves the program to the disk.
REV Display/Edit program revision notes.
ERROR{StartingLine} Merges the error routine interface and replaces any existing lines in that range. The default StartingLine of 8000 will be used if not specified.
ArrowUp Allows editing of previous command.
VDTxxxxCCpppps Creates a PRINT statement at line xxxx based on a VDT format stored in company code CC, with name pppp and VDT suffix s.
prog+ArrowUp Loads prog in program editor.   Note is does not save the existing program first.
prog+ArrowDn Runs prog.   Does not save first.
F1 Hotkey - allows you to temporarily run another application and return to the program being edited.
F2 Insert context sensitive text (Code completion).
