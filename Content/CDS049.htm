CDS049

Multiple File Verified Entries

 Last Revised: 12/22/09

This routine can be called to prompt the operator to enter one or more file verified entries.  For example, CDS049 could be used to enter a series of state/country codes, product classes, salesman numbers, etc.

All entries take place within a window displayed by CDS049.

Calling Format:

CALL "CDS049", MN, MX, XP, YP, TITLE$, TYP$, OUT$, FILNM$, FLD, OFF, LGTH {, PRFX$}

CDS049
MN minimum value or length
MX maximum value or length
XP window column position
If XP and YP are passed as zero, CDS049 will determine window position
YP

window row position
If XP and YP are passed as zero, CDS049 will determine window position
PROMPT$ prompt used for entry, and window title followed by " Selection"
TYP$ The entry type can be either a simple CDS069 data entry type such as A, L, R, 000, etc., or a complete CDS069 data entry type including verification parameters such as PFA{DYSW03,NAME,,,R}.  When TYP$ includes all verification parameters, then the arguments FILNAM$, FLD, OFF, and PRFX$ are not used.
DTA$ Returned data. If passed to CDS049, then the initial window will display the passed data. All entries made will be concatenated together to form DTA$.  DTA$ does not include any related field displayed.
FILNM$ Name of the file for verification.  Not used when TYP$ contains file name.
FLD The field number of the related field to display.  Only applies to traditional field number based records.  Use complete version of TYP$ when using string templates.
OFF The starting position or offset for the related field to be displayed.  Only applies to traditional field number based records.  Use complete version of TYP$ when using string templates.
LGTH Maximum number of characters to display of the related field.  This is also used to determine the size of the entry window.
PRFX$ A prefix to the key in the related file which would be required when the entered data is not the only key segment of the file.  Not used when TYP$ contains all verification parameters.

Since the parameters to CDS049 parallel those of CDS069, refer to the documentation on CDS069 for additional details.

Examples
To enter one or more state/country codes
Traditional field based file where the state name is in field 2, beginning at position 1, for a length of 27 characters.
CALL "CDS049",2,2,0,0,"State/Country Code","A",DTA$,"DISM00",2,1,27
Template based file where state name is NAME.

CALL "CDS049",2,2,0,0,"State/Country Code","PFA{DYSW03,NAME,,,R}",DTA$,"",0,0,27
To enter one or more product classes
Traditional field based file where the product class description is in field 2, beginning at position 1, for a length of 30 characters.

CALL "CDS049",1,3,0,0,"Product Class","R",OUT$,"XXUD27",2,1,30
Template based file where the Product Class Description DESC.

CALL "CDS049",1,3,0,0,"Product Class","PFR{XXIM27,DESC,,,R}",OUT$,"",0,0,30
