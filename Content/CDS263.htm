CDS263

Fax Dial String Processor

 Last Revised: 04/09/97

This program is used by the VSI*FAX fax scheduler to modify the dial string when the "dialcvt" program provided with VSI*FAX does not provide sufficient conversion.

It is typically needed when there is a authorization code required on long distance calls that follows the telephone number.

This program is not typically called from within an application program. Instead, it works like dialcvt using stdin and stdout.

This program is run by a script provided in the CDI/bin directory named dialcvt.example. Copy this script to CDI/bin/dialcvt. Then edit this script to specify the environment variables used:

CDS263
OUTSIDE the numbers dialed to obtain an outside line
LDPFX the numbers dialed prior to dialing a long distance number
LDSFX the numbers dialed after dialing a long distance number


If the LDPFX or LDSFX is not required, then you can leave the assignment line in the file with nothing to the right of the equal sign as in: LDSFX=

Comma's may be inserted as required to indicate a pause.

Note that the LDPFX and LDSFX codes will not be used for 1-800 and 1-888 no charge long distance calls.

Then edit the CDI/bin/faxenv file to specify the name of the dial conversion program. To use the standard dialcvt provided by V-Systems, use: FXDIALER=dialcvt

To use this dial conversion program provided with Dynamo, use: FXDIALER=$SMSDIR/bin/dialcvt
